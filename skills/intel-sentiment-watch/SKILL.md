# intel-sentiment-watch

## 执行目标
- 30-60 分钟内完成一次“可决策”的舆情快照。
- 输出：风险告警、情绪分布、高影响帖子、建议动作。

## 数据源（手动优先）
1. X / Twitter（关键词 + 品牌名 + 竞品名）
2. Reddit（行业子版块 + 品牌词）
3. 新闻/RSS（行业媒体、投放媒体、平台政策）
4. 可选：TikTok 趋势与评论摘要（仅作补充）

## 标准流程（手动）
1. 明确本轮监控范围
   - 品牌词（主品牌/产品名/常见拼写）
   - 风险词（scam、ban、refund、policy violation 等）
   - 时间窗（默认 24h，可切 7d）
2. 抓取样本并去重
   - 每源先取 Top 20-50 条高相关内容
   - 记录：来源、链接、时间、互动量、摘要
3. 情绪分层
   - 正面 / 中性 / 负面
   - 负面再标注：客服问题、产品问题、投放问题、政策问题
4. 风险分级
   - P1：高传播负面、潜在公关危机
   - P2：持续负面趋势、集中投诉
   - P3：普通吐槽、低传播
5. 产出结论
   - Top 风险 3 条
   - Top 机会 3 条
   - 今日建议动作 3 条（必须可执行）

- 监控对象：
- 时间窗口：
- 总样本量：
- 情绪分布：正/中/负（%）
- 负面主题 Top3：
- 高影响帖子 Top5（含链接）：
- 告警：P1/P2/P3
- 建议动作（今日）：

## 质量要求
- 不凭单条内容下结论；至少 2 个来源交叉验证。
- 负面结论必须带链接证据。
- 区分“真实用户投诉”与“营销号/机器人噪音”。
